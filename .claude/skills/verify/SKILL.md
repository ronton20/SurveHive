# Verifying SurveHive changes

Unity 6000.5.2f1 project; editor binary:
`/Applications/Unity/Hub/Editor/6000.5.2f1/Unity.app/Contents/MacOS/Unity`

Check first that the Unity editor doesn't have the project open
(`ls Temp/UnityLockfile`) — batchmode needs the lock.

## Layers (run in this order)

1. **Builder pass(es)** — scene/assets are generated by idempotent editor
   passes; re-run the relevant one after changing builder code:
   `Unity -batchmode -quit -projectPath . -executeMethod SurveHive.BuildTools.<Builder>.Apply -logFile build.log`
2. **Scene validator** (static wiring checks, ~200 asserts):
   `... -executeMethod SurveHive.BuildTools.BeehiveSceneValidator.Validate -logFile validate.log`
   Grep the log for `[FAIL]` and `validation PASSED`.
3. **Tests**: `Unity -runTests -batchmode -projectPath . -testPlatform <EditMode|PlayMode> -testResults r.xml -logFile t.log`
   (PlayMode smoke test boots the real Beehive scene and plays it.)
4. **Visual drive with screenshots** (the real surface):
   `Unity -projectPath . -executeMethod SurveHive.BuildTools.PlayModeVerifyDriver.Run -logFile drive.log`
   — **no `-batchmode`** (the game view must render; opens the editor GUI
   for ~1 min). It auto-plays the Beehive scene, force-equips the active
   skills, triggers a level-up offer, saves screenshots to `VerifyShots/`
   (gitignored), and quits. Edit its staged `OnEditorUpdate` switch to
   drive whatever the change under test needs.

## Gotchas

- Exit code 2 from `-runTests` = test failures; parse the results XML.
- Game-time waits in PlayMode tests hang if the game pauses
  (`GamePause` sets `timeScale = 0` on level-up/death) — advance with
  unscaled-time caps and click through level-up offers (see
  `BeehiveSmokeTest.RunGameSeconds`).
- Screenshots while paused capture a stale framebuffer unless the driver
  calls `EditorApplication.QueuePlayerLoopUpdate()` every editor update.
- TMP components created in code default to NoWrap (project TMP settings)
  — long UI text needs `textWrappingMode = TextWrappingModes.Normal`.
- Unity batch runs take 1–3 min each; run them in the background.
